## The build at a glance

The chat agent ships with **two engines in one site**:

1. **Live engine** — when an Anthropic API key is configured, the `/api/chat`
   route sends questions to Claude with your full knowledge base as context.
   Real, fluent, reasoning answers. This is the resume-bullet version.
2. **Self-contained fallback** — when no key is set (or the API fails, or the
   rate limit is hit), the chat runs a keyword-retrieval engine *in the
   browser* over the same knowledge base. Always works, costs nothing, can't
   break.

The site auto-detects which mode it is in and shows the user a small badge
("Live AI" / "Offline"). You get a working demo **today** and the full
LLM-powered version the moment you add a key — no rebuild.

Everything about you lives in **one file**: `lib/profile.ts`. Edit that and the
whole site *and* the chat update together.

## Phase 1 — Make it yours  ·  ~1–2 hours

Open `lib/profile.ts`. Every placeholder is wrapped in `[[ double brackets ]]`.
Replace them all:

- [ ] `github`, `linkedin` — your real profile URLs
- [ ] `location` — city/state
- [ ] Velarro **role title** and **dates**
- [ ] Real **metrics** for projects (retrieval accuracy %, latency, repo links, demo links)
- [ ] University **grad year** and **major** (confirm "Computer Science")
- [ ] Any extra skills, languages, or projects you want surfaced
- [ ] Drop your resume PDF into `public/resume.pdf`
- [ ] Set `NEXT_PUBLIC_GITHUB_USERNAME` in `.env.local` so the live GitHub card works

Tip: also expand the `knowledgeChunks` array — every chunk you add is a question
the chat can answer well. The richer this is, the smarter both engines feel.

try the suggested questions. This is already a shippable demo.

- [ ] Site renders, all sections look right on desktop + mobile

---

## Phase 3 — Turn on the real AI  ·  ~30 min

1. Get an Anthropic API key at https://console.anthropic.com.
2. Copy `.env.example` to `.env.local` and set `ANTHROPIC_API_KEY`.
3. Restart `npm run dev`. The badge flips to **"Live AI"** — answers are now
   generated by Claude with your knowledge base as context.

- [ ] Live mode answers an off-script question well (e.g. "would he fit a
      devtools team?")
- [ ] Default model is cheap (Haiku) — see cost note below

---

## Phase 4 — Deploy  ·  ~30–45 min

1. Push the repo to GitHub (public — recruiters should see the code).
2. Import it at https://vercel.com — Vercel auto-detects Next.js.
3. In Vercel project settings → Environment Variables, add `ANTHROPIC_API_KEY`
   and `NEXT_PUBLIC_GITHUB_USERNAME`. (Never commit the key — `.gitignore`
   already excludes `.env.local`.)
4. Deploy. You get a `*.vercel.app` URL in ~2 minutes.
5. Buy a domain (`baavansh.dev` or similar, ~$12/yr) and point it at Vercel.

- [ ] Live site loads on the custom domain
- [ ] Chat works in live mode in production
- [ ] Put the URL on your resume, LinkedIn, and GitHub profile

## Phase 5 — The differentiator: tool use  ·  ~2–4 hours (later)

Most "chat portfolios" are stale RAG. Make yours **live**. The API route is
already structured for this — extend it so Claude can call tools:

- [ ] `getLatestCommits` — pull your most recent public GitHub commits on demand
- [ ] `getLocation` / `getStatus` — "is he available?", "where is he based?"
- [ ] (stretch) recent LinkedIn posts or blog posts

A basic version already ships: the **live GitHub activity card** on the page
fetches your public events client-side. Phase 5 is wiring that same idea into
the *chat* as a Claude tool call.

## Cost & safety notes

- The live chat uses **Claude Haiku** by default — fast and cheap (fractions of
  a cent per message). Override with the `CHAT_MODEL` env var.
- A public AI endpoint can be abused to run up a bill. The API route includes a
  **basic per-IP rate limiter**. For real traffic, consider Vercel's firewall
  or a stricter limiter, and set a **spend limit** in the Anthropic console.
- The offline fallback means the site never *breaks* if the key is missing,
  expired, or rate-limited — it just quietly drops to retrieval mode.
